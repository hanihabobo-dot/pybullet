Logic 4. Shadow Generation: Mathematics & Geometry

calculate_shadow_boxel

Line ~385

This function creates a Semantic Boxel representing the occluded volume behind an object. The logic solves two critical geometric problems: Separation (avoiding overlap) and Projection (limiting length).

Problem 1: Axis-Aligned Separation (Avoiding Overlap)

A naive approach (moving radius distance away) causes the shadow boxel (an AABB) to overlap the object boxel (also an AABB) by up to 50% when viewed diagonally. We need to find the exact shift vector to make them just touch.

The Math:

Let Cobj be the object center and Eobj be its half-extent vector.
Let D be the unit direction vector from Camera to Object.
We want to find a scalar k such that the shadow start center Cstart = Cobj + k * D places the shadow AABB just outside the object AABB.

For two AABBs to be disjoint, they must be separated along at least one axis (Separating Axis Theorem).
For a specific axis i (x, y, or z), the condition to "clear" the object's extent is:
|k * Di| ≥ 2 * Eobj, i

Thus, the minimum distance required to clear axis i is:
ki = (2 * Eobj, i) / |Di|

To minimize the gap between object and shadow, we choose the axis that is cleared first:
kopt = min(kx, ky, kz) 
# Implementation (Line ~405):
k_values = []
for i in range(3):
    if abs(direction[i]) > 1e-6:
        # Distance to clear axis i
        k = 2.0 * obj_boxel.extent[i] / abs(direction[i])
        k_values.append(k)
    else:
        # Moving parallel to this face, infinite distance to clear it
        k_values.append(float('inf'))

# Pick the first exit point
k_opt = min(k_values)
shadow_start = obj_pos + direction * k_opt * 1.01 # 1% buffer

Problem 2: Perspective Projection (Limiting Length)

Shadows shouldn't be infinite; they end where the sightline hits the table. We simulate this by projecting the object's center ray.

The Math:

Ray Equation: P(t) = Pcam + t * D
Plane Equation: Z = Htable (where Htable ≈ 0.325m)

Substitute Ray Z into Plane Z:
Pcam, z + t * Dz = Htable

Solve for t:
t = (Htable - Pcam, z) / Dz

Result: shadow_end = P(t). The Shadow Boxel is then constructed as the AABB bounding the swept volume from shadow_start to shadow_end.

Logic 5. Free Space Discretization: Octree Algorithm

generate_free_space

Line ~500

This function fills the empty "air" in the workspace with Boxels using a recursive spatial partitioning strategy (Octree), implemented iteratively via Breadth-First Search (BFS).

Why Octree?

We want large boxels for open spaces (efficient planning) and fine-grained boxels near obstacles (accurate collision avoidance). Octrees naturally provide this multi-resolution representation.

The Algorithm (Step-by-Step):

  1. Initialization:
    • Define Workspace AABB: ws_min to ws_max (Z: 0.325m to 0.825m).
    • Create Root Node covering the entire workspace.
    • Initialize Queue = [Root].
  2. BFS Loop (Layer by Layer):
    • For each Node in current layer:
      • Intersection Test: Check if Node.AABB overlaps with ANY Known Boxel (Object or Shadow).
      • Case 1: No Intersection (FREE)
        The node represents completely empty space. Add it to free_boxels list. Stop recursing for this branch.
      • Case 2: Intersection & Small (OCCUPIED)
        The node hits an obstacle and is smaller than min_size (5cm). Mark as occupied. Discard (we only want free space).
      • Case 3: Intersection & Large (MIXED)
        The node hits an obstacle but is large enough to contain gaps.
        Action: Split into 8 equal child octants. Add children to Next Layer Queue.
    • Visualization (If enabled): Draw newly found Free Boxels (Cyan) and next layer Candidates (Yellow). Wait 1s.

# AABB vs AABB Intersection Test
# Two boxes overlap if and only if they overlap on ALL 3 axes.
def intersects(box_a, box_b):
    return (box_a.max_x >= box_b.min_x and box_a.min_x <= box_b.max_x) and
           (box_a.max_y >= box_b.min_y and box_a.min_y <= box_b.max_y) and
           (box_a.max_z >= box_b.min_z and box_a.min_z <= box_b.max_z)
